# FPV 四轴飞行器 3D 仿真器

## 三种输入模式

### 1. 键盘模式（默认，按 `1` 切换）

- 仿真器内部维护虚拟飞行姿态（roll/pitch/yaw/altitude）
- 用户通过键盘操控：W/S 俯仰、A/D 横滚、Q/E 偏航、空格/↑↓ 油门
- **上行**：MCU 持续发送真实 IMU 数据到仿真器，用于 HUD 左上角显示真实 IMU 数值
- **下行**：仿真器每 50ms 向 MCU 发送混控后的电机指令 `M:...`，驱动真实电机旋转

```
键盘输入 → 虚拟姿态 → 混控计算 → M:指令 → MCU → PWM → 电机
                                      │
真实 IMU ← MCU ← QMI8658               │
                                      │
         └── HUD 显示真实 IMU 数值 ◄──┘
```

**油门特性**：空格快速增加（120/s），↑ 慢增（50/s），↓ 快减（60/s），无操作缓慢衰减（8/s）。

### 2. 手控 IMU 模式（按 `M` 切换）

- 仿真器姿态完全由真实 IMU 数据驱动（手持 IMU 的姿态 = 屏幕中无人机的姿态）
- **下行**：仿真器计算负反馈修正量，发送 `M:` 指令驱动电机

```
手持 IMU 倾斜 → QMI8658 → MCU → I:数据 → 仿真器
                                          │
                                    姿态 = 真实 IMU
                                          │
                                    计算负反馈修正
                                    r = -roll × 1.8
                                    p = -pitch × 1.8
                                          │
电机 ← PWM ← MCU ← M:指令 ←──────────────┘
```

**核心逻辑**：当 IMU 向右侧倾斜（roll > 0），仿真器发送 `roll = -roll × 1.8`（负值），MCU 混控后右侧电机减速、左侧电机加速，产生向左的修正力矩——模拟飞控试图恢复水平。

**固定油门 30%**，偏航不修正（yaw = 0）。

### 3. 串口模式（按 `2` 切换）

- 仿真器姿态完全由串口数据驱动（纯显示模式）
- 不发送电机指令（`M:0,0,0,0`），电机不转

## 串口协议

### MCU → 仿真器（上行，125Hz）

```
I:ax,ay,az,gx,gy,gz\r\n
```

| 字段 | 含义 | 单位 | 说明 |
|------|------|------|------|
| `ax` | X 轴加速度 | 0.01g | 原始值 = 实际值 × 100 |
| `ay` | Y 轴加速度 | 0.01g | |
| `az` | Z 轴加速度 | 0.01g | |
| `gx` | X 轴角速度 | 0.01°/s | 原始值 = 实际值 × 100 |
| `gy` | Y 轴角速度 | 0.01°/s | |
| `gz` | Z 轴角速度 | 0.01°/s | |

示例：`I:-15,3,-102,12,-8,2\r\n`

#### 适配其他 IMU

仿真器不关心 IMU 型号，只要通过串口（115200bps）按上述格式发送文本行即可接入。适配时注意：

- **坐标系**：使用 NED 系（前-右-下），即 X=机头方向、Y=右侧、Z=下方
- **加速度**：发送 `实际加速度(g) × 100` 的整数值，水平静置时 `az ≈ 100`
- **陀螺仪**：发送 `角速度(°/s) × 100` 的整数值，符号遵循右手定则
- **行尾**：`\r\n`
- 仿真器会自动识别 `I:` 前缀并切换到原始 IMU 模式，无需额外配置

### 仿真器 → MCU（下行，20Hz）

```
M:throttle,roll,pitch,yaw\r\n
```

| 字段 | 含义 | 范围 | 说明 |
|------|------|------|------|
| `throttle` | 油门 | 0~100 | 基础油门量 |
| `roll` | 横滚修正 | ±100 | 正值 = 向右横滚的修正力 |
| `pitch` | 俯仰修正 | ±100 | 正值 = 向前俯仰的修正力 |
| `yaw` | 偏航修正 | ±100 | 正值 = 顺时针偏航的修正力 |

示例：`M:30,-15,10,0\r\n`

## MCU 端的混控计算（X 构型）

仿真器下发的 `throttle/roll/pitch/yaw` 在 MCU 上按以下公式分配：

```
M1 (前右) = throttle + pitch + roll + yaw
M2 (前左) = throttle + pitch - roll - yaw
M3 (后左) = throttle - pitch - roll + yaw
M4 (后右) = throttle - pitch + roll - yaw
```

结果钳位到 0~100，输出为 PWM 占空比（1kHz）。

**直观理解**：
- **横滚修正** (roll)：正值 → M1↑ M3↑ / M2↓ M4↓ → 左侧电机更强 → 飞机向右滚转
- **俯仰修正** (pitch)：正值 → M1↑ M2↑ / M3↓ M4↓ → 前侧电机更强 → 飞机向后仰
- **偏航修正** (yaw)：正值 → M1↑ M3↑ / M2↓ M4↓ → 对角 M1/M3 加速产生顺时针扭矩

## 仿真器中的 IMU 数据处理

### 互补滤波器

仿真器收到 MCU 的加速度计+陀螺仪原始数据后，使用互补滤波器融合：

```
姿态角 = 0.98 × 陀螺仪积分角 + 0.02 × 加速度计推算角
```

- 加速度计提供低频无漂移的绝对角度（通过重力方向推算 roll/pitch）
- 陀螺仪提供高频响应快的角速度积分
- 互补滤波结合两者优势：响应快 + 不漂移

### 坐标系映射与符号约定（关键）

QMI8658 与仿真器使用不同的坐标和符号约定，MCU 固件在发送 `I:` 数据前必须完成映射。以下约定经过硬件验证，**不可随意修改**。

#### 坐标系定义

| 对象 | X 轴 | Y 轴 | Z 轴 |
|------|------|------|------|
| QMI8658 芯片 | 右 | **前** | 上 |
| 仿真器机载系 | 右 | — | 上（机头 = +Z） |

QMI8658 的 Y 轴（前）对应仿真器俯仰轴，X 轴（右）对应仿真器滚转轴。

#### 物理约定差异

| 项目 | QMI8658 | 仿真器虚拟 IMU |
|------|---------|---------------|
| 加速度计 | 测量**支撑力**（水平时 Z=+1g） | 虚拟模型 `ax=G·sin(pitch)` |
| 陀螺仪 | **右手定则**（正转=逆时针沿轴看） | 角速率正=角度增加 |

由于 QMI8658 测量的是支撑力（非重力），前倾时机头端支撑力投影为 **-Y**（负值）；而仿真器虚拟模型 `ax = G·sin(pitch)` 在前倾时产生正值。陀螺仪遵循右手定则，绕 X 轴正转 = 机头上仰，与仿真器"前倾=正"的约定相反。

#### MCU 固件中的最终映射（main.c）

```
原始 QMI8658 输出                        仿真器 I: 字段
──────────────────────────────────       ──────────────
acc[1] (Y轴, m/s²)  ──× (-1)──►  ax  (俯仰加速度, 0.01g)
acc[0] (X轴, m/s²)  ──× (+1)──►  ay  (滚转加速度, 0.01g)
acc[2] (Z轴, m/s²)  ──× (-1)──►  az  (垂直加速度, 0.01g)
gyro[1] (绕Y, rad/s) ──× (+1)──►  gx  (滚转角速率, 0.01°/s)
gyro[0] (绕X, rad/s) ──× (-1)──►  gy  (俯仰角速率, 0.01°/s)
gyro[2] (绕Z, rad/s) ──× (+1)──►  gz  (偏航角速率, 0.01°/s)
```

即代码中的实现：

```c
int ax = (int)(-acc[1] * 100.0f);         // 俯仰加速度：取反
int ay = (int)( acc[0] * 100.0f);         // 滚转加速度：不变
int az = (int)(-acc[2] * 100.0f);         // 垂直加速度：取反
int gx = (int)( gyro[1] * 5729.578f);     // 滚转陀螺：不变
int gy = (int)(-gyro[0] * 5729.578f);     // 俯仰陀螺：取反
int gz = (int)( gyro[2] * 5729.578f);     // 偏航陀螺：不变
```

**缩放系数说明**：
- 加速度：`× 100` 将 m/s² 转为 0.01g（9.81 m/s² ≈ 100 单位）
- 陀螺仪：`× 5729.578`（= 100 × 180/π）将 rad/s 转为 0.01°/s

#### 验证标准

手持 IMU 做以下动作，仿真器应实时正确响应：

| 手持动作 | 仿真器应显示 |
|---------|------------|
| 向前倾倒（机头下倾） | 俯仰角正值（前倾/低头） |
| 向后倾倒（机头上仰） | 俯仰角负值（后仰/抬头） |
| 向右倾斜（右翼下沉） | 滚转角正值（右倾） |
| 向左倾斜（左翼下沉） | 滚转角负值（左倾） |

### 偏航死区

偏航角速度低于 1.5°/s 时视为零漂，不积分，防止静止时的偏航漂移。
